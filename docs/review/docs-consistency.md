# app-harness — Documentation Consistency Audit

**Scope:** Cross-checks every shipped doc (`SKILL.md`, `RUBRIC.md`, `docs/DESIGN.md`, `docs/ADAPTER-CONTRACT.md`, `scripts/README.md`, `scripts/CONTRACT.md`, `CHANGELOG.md`) against the real implementation — `harness.workflow.js` (behavioral ground truth), `scripts/harness.sh` (dispatcher), the on-disk `scripts/` and `adapters/*/` layout, and all seven `adapters/*/rubric.md` profiles.
**Method:** Read each doc in full, read the workflow and dispatcher source, enumerated `scripts/` and `adapters/*/` on disk, read every adapter rubric and two adapter manifests, then compared claim-by-claim. Rows are sorted High → Medium → Low, with accurate/no-drift rows grouped last to show full coverage.

## Summary

**Total drift rows: 11** (excluding 9 accurate/no-drift coverage rows).

- **High: 3** — (1) `SKILL.md` "Helper Scripts" claims per-platform scripts are "not flat in scripts/" while `scripts/gate.sh` et al. remain; (2) `scripts/README.md` claims the dir "keeps only shared pieces" while 5 legacy web-only flat scripts persist; (3) `SKILL.md` "After It Returns" omits the returned `screenshots` field.
- **Medium: 5** — stale `scripts/gate.sh` gate reference in `RUBRIC.md`; incomplete dispatcher verb lists in `SKILL.md` (×2 spots, missing `detect`+`rubric`) and `DESIGN.md` (missing `rubric`); `DESIGN.md` scripts tree omits the legacy flat scripts; `scripts/CONTRACT.md` omits them too.
- **Low: 3** — 4-vs-5 phase count (missing Preview) in `SKILL.md`; undocumented `adapters/ai-service/lib/`; stale "implementation in progress" status + missing changelog deprecation note (grouped).

**Highest-impact, most-pervasive drift:** the five legacy web-only flat scripts (`scripts/{gate,boot,probe,preview}.sh`, `scripts/slop-scan.mjs`) still exist on disk, yet `SKILL.md`, `README.md`, `DESIGN.md`, and `CONTRACT.md` each either explicitly claim they were relocated into `adapters/<id>/` or omit them entirely — actively misleading any maintainer about the actual `scripts/` layout and leaving dead code undocumented. Also notable: `RUBRIC.md` still points adapter authors at the stale `scripts/gate.sh` as "the hard GATE" when the real gate is `harness.sh gate` → `adapters/<id>/gate.sh`.
